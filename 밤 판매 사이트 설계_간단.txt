홈페이지 구성
- 메뉴는 상품 구매, 주문 현황(관리자만 보이도록한다.), 마이페이지(로그인시만 보이도록한다.), 로그인(로그아웃)으로 구성한다. 아래는 각 페이지 설명이다.
- 상품 구매 : 상품들이 카드식으로 사진과 설명으로 구성되고 서버에 저장된다. 장바구니 담기, 주문 버튼이 있으며, 관리자만 편집을 할 수 있다. 배송방법도 설명한다. 주문 하기시 주문 페이지로 이동한다. 
    하단에 저장 방법 : 김치 냉장고가 최고의 저장 방법임을 소개, 보관 시 주의 사항, 밤 상태 확인 방법, 밤 활용법, FAQ 사례를 표시한다. 
    농장 연락처와, 위치도 하단에 표시한다.
    [주문 페이지]는 주문시 나타나며, 회원 정보를 자동으로 입력칸에 채워준다. 취문 취소 또는 변경도 가능해야 한다.
    수취인 정보칸에는 주문자와 동일 체크박스를 만들어서 체크되면 주문자 정보를 그대로 입력해 준다. 
    기본적인 주문 요약과, 배송안내, 결제 안내(무통장 입금만 가능하도록 안내), 농장 연락처등을 함께 보여주고 안내 카드는 관리자가 이것을 편집할 수 있고 서버에 저장되어야 한다.
- 주문 현황 : 관리자가 로그인한 경우에만 메뉴가 나타나야 하고 일반 계정은 반드시 접근이 불가해야 한다. 주문 현황 페이지에는 아래 주문 목록, 통계, 메모를 포함한다. 자세한 각 설명은 아래 설명을 참고한다.
  [주문 목록] 상단에 주문 목록이 엑셀 형식의 리스트로 제공(항목은 차례대로 일자, 주문자/수취인, 주소, 상품, 수량, 금액, 상태, 비고, 주문번호, 편집)되며 서버에 저장되어야 한다.
            일자 : 한국 날짜 시간 기준으로, YY.MM.DD로 나타낸다.
            주문자/수취인, 주소 : 3줄로 첫줄에는 주문자와 연락처 그리고 둘째줄에는 수취인과 연락처가 나타난다. 셋째줄에는 주소가 나타난다.(주소는 한줄당 한글 10자씩 보여준다.)
            상품 : 한줄당 한글 5자씩 보여준다.
            금액 : 숫자를 만원단위로 표현해준다. 예를 들면 10000원 = 1만원, 150000원 = 15만원
            상태 : 기본은 입금 대기로 표시하고, 관리자가 드롭다운 형식으로 입금 완료, 출고 완료, 교환 완료, 환불 완료를 선택할 수 있게 한다. 각 선택에 따라 라인 전체에 색상이 변경되어야 한다.(입금 대기는 녹색, 출고 완료는 회색, 교환 완료와 환불 완료는 연노란색)
            비고 : 한줄당 한글 10자씩 보여준다.
            주문번호 : 손님이 구매한 주문번호는 GYYMMDD시분초로, 관리자가 직접 입력한 주문번호는 AYYMMDD시분초 형태로 자동으로 부여한다. YYMMDD다음에 자동 줄바꿈을 한다.
            리스트 하단에 페이지당 나타낼 수 있는 갯수를 선택 가능 하도록 한다. (10개, 20개, 30개, 50개, 100개)
            주문 추가 버튼을 만들어 관리자가 직접 추가 할 수 있게 한다.
            리스트 상단에 주문 검색 필터를 만든다. (일자, 상품, 주문자, 수취인, 상태)
            리스트 상단에 엑셀 추출 버튼을 만든다. (엑셀 추출시 3줄이었던 주문자/수취인,주소를 3열을 추가하여 항목에 맞게 한줄로 고쳐 모두 한줄에 표시되도록 한다.)
  [통계] 주문 통계를 한눈에 파악하기 좋게 만든다.
  [메모] 공지사항 게시판과 동일한 형태로 관리자만 사용가능 하고 서버에 저장되도록 만든다.
- 마이페이지 : 내정보와 구매 내역및 회원 탈퇴와 정보 수정이 보이도록 한다. 
    구매내역에 주문한 내역이 요약되어 보여지고, 현재 상태를 알려준다 관리자가 주문 현황에서 상태를 변경하면 입금 완료시 입금 확인 완료, 출고 완료등을 실시간으로 표시해준다. 
    하단에는 입금 확인 안내 사항 카드를 배치하여 "입금 대기 상태는 입금 완료 후 24시간 체크 소요 시간이 발생 할 수 있습니다. 확인 후 입금 확인 완료로 표기 됩니다." 라는 주의 사항을 표시해준다.
